# 狐Tone (KituneTone)

<p align="center">
  <img src=".github/assets/kitsunetone-logo.png" alt="狐Toneロゴ" width="480">
</p>

**狐Tone** は、オープンソース[LMMS](https://lmms.io/) をフォークして生まれた、独自拡張を持つオリジナルDAWです。
VST3,ARA2,マルチトラック録音、日本語環境向けの改善などを独自に追加しています。

> [!IMPORTANT]
> 狐ToneはLMMSから派生した非公式フォークです。LMMSプロジェクトによる公式製品ではなく、LMMSの開発チームによる承認・サポートを受けたものでもありません。

> [!WARNING]
> 現在は開発段階です。完全な動作は保証されません。

## ダウンロード

Windows 10 / 11（64bit）向けのビルドは、[Releases](https://github.com/SakiikaVR/KituneTone-DAW/releases) から入手できます。
配布アーカイブを任意の場所へ展開し、`KitsuneTone.exe` を実行してください。インストールは不要です。
- 出力デバイスは設定画面（オーディオ → PortAudio → Input device）で設定できます。

VST3プラグインは、`C:\Program Files\Common Files\VST3` などの標準フォルダーから自動検出されます。

> [!NOTE]
> 低遅延で使う場合は、設定画面の「Windows WASAPI」をお使いください。ASIOは非対応です。
> **排他モード（低遅延）** を有効にすることを推奨します。

## 狐Toneの主な機能

### ネイティブVST3ホスティング

- VST3インストゥルメントとVST3エフェクトをプロセス内でホスト
- ノート、ベロシティ、CC、ピッチベンド、アフタータッチをVST3イベントとして送信
- プラグインの状態をプロジェクトに保存
- プラグイン本来のGUIを表示（プラグインが報告するサイズ・リサイズに追従）
- **VST3パラメーターのオートメーション** — プラグインGUI上で目的のツマミを右クリックし「オートメーションを有効化」すると、そのパラメーターをLMMSのオートメーションで制御できます（Vitalなど大量のパラメーターを持つプラグインでも、使いたいツマミだけを個別に対象化できます）

VST3インストゥルメントは、インストゥルメント一覧の「VST3」からトラックへ追加できます。
VST3エフェクトは、エフェクトチェインの「エフェクトを追加」から選択できます。

### 録音（マルチトラック録音）

トランスポートの録音ボタン（●）を押すと、**録音待機（アーム）したトラックへ同時に録音**します。
録音は再生と一体化しており、押した瞬間の再生位置からそのまま録音が始まります。

- **トラックごとの録音待機ボタン** — 各トラックの操作エリアにある●ボタンで、そのトラックを録音対象にします
- **サンプルトラック → マイク録音** — アームしたサンプルトラックへオーディオ入力を録音。トラックごとに使用する入力デバイス（マイク）を選べます（バックエンドに応じてデバイス一覧をフィルタ）
- **インストゥルメントトラック → MIDI録音** — アームしたインストゥルメントトラックへ、MIDIキーボードなどの演奏をノートとして録音します
- **途中から録音（パンチイン）** — 再生位置（プレイヘッド）の途中から録音を開始できます（常に0秒からになりません）
- **録音中のリアルタイム表示** — 録音しているクリップが、停止を待たずにトラック上で伸びていく様子をその場で確認できます
- **録音レイテンシー補正** — 設定画面で録音の遅延補正量を指定でき、入力の時間ずれを吸収します
- 録音停止後はWAVファイル（マイク録音）として保存され、それぞれのトラックへ配置されます
- 入力デバイスは設定画面（オーディオ → PortAudio → Input device）でも既定を選べます
- 再生と同じデュプレックスストリームを使うため、排他モードのデバイスでもそのまま録音できます
- 録音ファイルは、ユーザーのサンプルフォルダー内にある `recordings/` へ保存されます

## ARA２の使い方

1. サンプルトラックにオーディオクリップを配置します。
2. そのトラックのエフェクトチェインへARA対応VST3エフェクトを追加します。
3. エフェクトの「UI」ボタンからプラグイン画面を開きます。
4. プラグイン側に波形が表示されたことを確認し、狐Tone側で再生します。

プラグインを追加した後にクリップを配置した場合は、プラグインUIを一度閉じてから開き直すと再取り込みされます。
診断情報は、通常 `ドキュメント\lmms\lmms_ara_debug.log` に出力されます。

### タイムストレッチ（ピッチ保持）

オーディオ素材の再生速度を、**ピッチ（音の高さ）を保ったまま**変更できます。WSOLAアルゴリズムによる高品質な伸縮です。

- サンプルクリップを右クリック →「タイムストレッチ」からダイアログを開きます
- **2倍 / 1/2倍 / 等倍**のワンタッチボタンに加え、任意の倍率を指定できます
- **BPM指定** — 素材の元BPMを入力すると、プロジェクトのテンポに自動で合わせて伸縮します
- クリップを複製・貼り付けしても伸縮が二重に掛からないよう、元の音声を保持したまま処理します

### 低遅延オーディオ

- オーディオインターフェイスは **PortAudio** に一本化（WASAPI / DirectSound / MME）
- **WASAPI排他モード**によるASIOに近い低遅延再生（設定でオン・オフ、非対応デバイスでは自動的に共有モードへフォールバック）
- WASAPIストリームは常にMMCSS「Pro Audio」優先度で動作
- サンプルレート不一致時はWASAPIの自動レート変換で吸収

### MIDIルーティング

- VST3プラグインが生成するMIDI（アルペジエーター、MIDIジェネレーターなど）をトラックのMIDI出力として取り出せます
- **内部MIDIバス**: MIDI出力を有効にしたトラックから、MIDI入力を有効にしたトラックへ、狐Tone内部で直接ルーティングできます（loopMIDIなどの仮想MIDIケーブルは不要）
- チャンネル設定（CHAN）で送り先をフィルタでき、外部MIDIデバイスへの出力とも併用できます

MIDIジェネレーター系VST3 → シンセVST3のような構成は、両トラックのピアノタブで
送信側の「MIDI出力を有効にする」と受信側の「MIDI入力を有効にする」をオンにするだけでつながります。

### ウィンドウシステム

- ソングエディタはワークスペースに常時全面表示される固定パネル（閉じる・移動・リサイズ不可）
- その他のウィンドウ（ピアノロール、ミキサー、プラグインGUIなど）はすべて狐Toneスタイルの浮動ウィンドウ
- 各浮動ウィンドウは画鋲ボタンで**常に最前面**へピン留め可能

### ARA 2ホスティング（実験的）

MelodyneやVoviousなどのARA対応VST3プラグインを、サンプルトラックのエフェクトとしてホストできます。

- トラック上のオーディオクリップをARAオーディオソースとして自動登録
- クリップの追加・移動・リサイズ・差し替えを検知して自動で再取り込み（プラグインを挿す順番・音声を置く順番のどちらでも動作）
- クリップ全体へのランダムアクセス（プラグインが必要な範囲を任意に読み出し）
- DAWのテンポと拍子をmusical contextとして連携（「Use DAW Tempo / Beat」）
- 再生・停止・再生位置をDAW ⇔ プラグイン双方向で同期
- トラック内の複数クリップを個別のARAリージョンとして配置
- プレイバックレンダラー / エディタレンダラー / エディタビューの3ロールをバインド
- **編集内容の永続化** — プラグイン内で行った編集（ピッチ補正など）をプロジェクトに保存し、開き直したときに復元します（ARA 2の部分永続化を使用、対応プラグインのみ）

ARA連携は実験的な実装です。基本的な音声取り込み、波形表示、解析、テンポ・再生同期、編集内容の保存・復元に対応しています。
動作はプラグインによって異なる場合があります（編集の永続化はプラグインがARA 2の部分永続化に対応している必要があります）。

### ファイルのドラッグ＆ドロップ

Windowsのエクスプローラーなどから、WAV、MP3、Ogg Vorbis、FLACなどのオーディオファイルを
サンプルトラックへ直接配置できます。複数ファイルの同時ドロップにも対応しています。

**MIDIファイルのドラッグ＆ドロップ** にも対応しました。エクスプローラーからソングエディタのトラック領域へ
`.mid` / `.midi` ファイルをドロップできます。

- **ドロップした位置に配置** — 落とした横位置（小節）に合わせてノートが置かれます
- **既存の楽器で鳴らす** — インストゥルメントトラックの上にドロップすると、新しいトラックを作らずにそのトラックの楽器で演奏します（新規トラックにドロップした場合の既定音源は TripleOscillator）
- **ベロシティなどを保持** — 楽器が変わっても、各ノートのベロシティ・パン・長さはそのまま引き継がれます
- **オートメーションの取り込みを確認** — ドロップ時にダイアログが表示され、テンポ（BPM）・CC・ピッチベンドなどのオートメーションを取り込むかどうかを選べます

### 日本語環境とUIの改善

- UIフォントをNoto Sans / Noto Sans JPへ統一
- 設定画面を含むUIの日本語翻訳を整備（同梱ロケールは日本語のみ、英語は標準表示）
- 狐Toneで追加した機能の日本語表示
- Windows向け最適化（1msタイマー精度、プロセス優先度、ネイティブビュー埋め込み時の描画最適化）

### 構成の整理

このフォークでは、安定性とWindowsへの最適化を優先して次の構成にしています。

- Carla、Carla Rack、Carla Patchbayをビルド対象から除外
- SDLオーディオを除外し、オーディオインターフェイスをPortAudioに一本化
- VSTプラグインGUIの埋め込みはQt方式のみ（ネイティブ埋め込み・XEmbedは廃止）

## ソースからビルドする

### 必要なもの

- Visual Studio 2022（C++によるデスクトップ開発）
- [Qt 6.8.x](https://www.qt.io/)（`msvc2022_64`）
- [vcpkg](https://github.com/microsoft/vcpkg)
- CMakeとNinja

### ビルド例

Visual Studioのx64 Native Tools Command Promptで実行してください。Qtとvcpkgのパスは環境に合わせて変更します。

```bat
vcpkg install --triplet x64-windows

cmake -S . -B build -G Ninja ^
  -DCMAKE_BUILD_TYPE=RelWithDebInfo -DWANT_QT6=ON -DWANT_SDL=OFF ^
  -DCMAKE_PREFIX_PATH=C:/Qt/6.8.3/msvc2022_64 ^
  -DCMAKE_TOOLCHAIN_FILE=C:/vcpkg/scripts/buildsystems/vcpkg.cmake ^
  -DVCPKG_TARGET_TRIPLET=x64-windows ^
  -DCMAKE_CXX_FLAGS_RELWITHDEBINFO="/MD /Zi /O2 /Ob1 /DNDEBUG"

cmake --build build
```

ビルドされる実行ファイルは `build/KitsuneTone.exe` です。

LMMS本来のビルド情報については、[LMMSのビルド手順](https://github.com/LMMS/lmms/wiki/Compiling) も参照してください。
